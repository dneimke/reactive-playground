Observables

This repository is a place for me to learn about Rxjs and using Observables.

Why Observables?

The broad adoption of web platforms and frameworks such as Angular and Firebase, has meant that Observables are fast becoming the preferred approach to dealing with data sources. Angular and Firebase both use Rxjs as their Observable implementation for the web. The flow-on effect means that we are now seeing that adoption spreading more broadly throughout the component ecosystem.

Videogular2 - a popular Angular module for working with Video uses a reactive approach which is also based on Rxjs. New frameworks which build on top of Rxjs such as NGRX are also coming to the prominence.

What are Observables?

The ReactiveX documentation uses the following definition for Observables:

An observer subscribes to an Observable. An Observable emits items or sends notifications to its observers by calling the observersâ€™ methods.

An Observer implements 3 methods which the Observable can call into:

  • next
  • error
  • complete

You can create a custom Observer and pass it to the subscribe method of an observable to process events within an observable source.
